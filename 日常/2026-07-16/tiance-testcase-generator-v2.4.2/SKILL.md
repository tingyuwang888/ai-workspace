v2.3.0 新增：内置 Excel 报告导出（export_excel），运行一次同时输出 JSON + Excel（16列蓝底白字表头、汇总+用例双Sheet、冻结首行+自动筛选），openpyxl 可选依赖。

## 前置条件

- Python 3.8+ 及 pandas、openpyxl (`pip install pandas openpyxl`)
- 用户提供策略部署落地方案 Excel 文件（.xlsx）

## 核心流程

### Step 1: 解析落地方案 Excel

运行解析脚本提取结构化数据：

```bash
python3 scripts/parse_strategy_excel.py "<Excel文件路径>" -o parsed_strategy.json
```

脚本从 Excel 提取以下 sheet（自动检测表头行）：

| Sheet | 提取内容 |
|-------|---------|
| 场景策略 | 策略列表（policyCode, policyName, bizType 等） |
| 规则集 | 规则集标识、适用业务场景、企业类型路由 |
| 规则 | 规则编号、表达式、命中结果、入参字段 |
| 函数 | 函数名称、计算逻辑、入参出参 |
| 字段管理 | 中文显示名 → 系统编码映射（182+ 字段） |
| 风险决策配置 | 风险等级名称、标识、信号等级 |
| 三方数据 | 接口名称、入参出参、ETL 逻辑 |

输出 JSON 包含 `summary` 统计和各维度结构化数据。每条规则的入参字段已自动映射为系统字段编码。

### Step 2: 分析解析结果

读取 `parsed_strategy.json`，向用户报告解析概要：
- 策略数量与列表
- 规则集数量及各集规则数
- 规则总数及命中结果分布（如高风险预警 82 条、中风险 51 条等）
- 函数数量
- 字段覆盖率（已映射 / 总入参数）

让用户确认是否继续，或需要调整范围（如只生成某几个规则集的用例）。

### Step 3: 生成测试用例

根据解析数据生成测试用例。每条用例格式：

```json
{
  "id": "TC_001",
  "desc": "用例描述",
  "group": "规则|函数|决策流分支覆盖|策略预警等级覆盖|跨子策略综合",
  "caseType": "正案例|反案例|边界案例",
  "scenario": "详细测试场景描述",
  "expected": "预期命中：规则编号(规则中文名)[风险等级] 或 未命中：规则编号(规则中文名)",
  "params": {
    "字段编码": "构造的测试值",
    "S_S_BIZID": "必填，业务ID",
    "S_S_CUSTNO": "必填，客户号",
    "S_S_ORGCODE": "必填，机构编码"
  },
  "targetRuleSet": "规则集标识",
  "targetRule": "规则编号（规则级用例）",
  "thirdPartyMock": {
    "interfaces": ["2003-企查查客户身份识别"],
    "fields": ["C_N_XXX", "C_O_XXX"],
    "mockHint": "mock 数据结构说明"
  }
}
```

#### 3.1 规则级用例生成策略

对每条规则，分析其表达式中的条件，生成以下用例：

**A. 正案例（命中）**：构造满足所有条件的参数值
**B. 反案例（未命中）**：至少一个条件不满足
**C. 边界值案例**：针对阈值条件生成临界值用例

表达式模式识别与参数构造规则：

| 表达式模式 | 含义 | 命中值示例 | 未命中值示例 | 边界值 |
|-----------|------|-----------|-------------|--------|
| `字段 >0 且 详情 <>空` | 计数>0且有详情 | 计数=1, 详情=[{...}] | 计数=0, 详情=[] | 计数=1（最小命中） |
| `字段 ≥ X%` | 百分比阈值 | X+10% | X-10% | X%（精确阈值） |
| `字段 >= N万` | 金额阈值 | N+100万 | N-100万 | N万（精确阈值） |
| `字段 >= N` | 整数阈值 | N+1 | N-1 | N（精确阈值） |
| `字段 < X%` | 百分比上限 | X-5% | X+5% | X%（精确阈值） |
| `字段 ≤ N天` | 天数上限 | N-1 | N+5 | N（精确阈值） |
| `字段 ≥ N年` | 年限阈值 | N+1年前 | N-1年前 | N年前（精确阈值） |
| `字段 = N` | 精确匹配（支持负数） | N | N+5 | N |
| `字段 为X或Y` | 枚举匹配（含"或"在值内） | X | 其他值 | Y |
| `字段 为/吊销/或/撤销/` | 斜杠分隔枚举 | 吊销 | 正常 | 撤销 |
| `字段 为是` | 布尔匹配 | "是" | "否" | - |
| `A 或 B` / `A || B` | 复合或 | A满足即可 | A和B都不满足 | 仅A/仅B |
| `A 且 B` | 复合且 | A和B都满足 | 仅A满足 | - |

**D. AND 条件拆解**（v2.0）：对含多条件 AND 的规则，逐一令单个条件不满足，其余满足，验证不命中
**E. OR 条件拆解**（v2.0+）：对含 OR（"或"或"||"）的规则，分别仅满足每个分支，加全不满足的反案例和分支边界值。正确处理括号嵌套分组。
**F. 严格运算符边界**（v2.0）：对 `>` / `<`（不含等于）生成恰好等于阈值的"不命中"用例

```json
// 变更记录类详情
[{"ChangeDate": "近30天内日期", "ProjectName": "匹配关键字", "BeforeList": "变更前", "AfterList": "变更后"}]

// 失信/被执行记录类详情
[{"Liandate": "近30天内日期", "Anno": "公告内容", "ExecuteGov": "执行法院", "ExecuteStatus": "执行状态"}]

// 司法案件类详情
[{"CaseRole": "被告", "CaseAmount": 涉案金额, "CaseType": "借款合同纠纷"}]
```

#### 3.2 函数级用例生成策略（v2.1 增强）

对每个函数，根据计算逻辑和入参出参生成用例。v2.1 新增关键词模糊匹配回退——当函数名与模板不完全一致时，通过关键词（如"初始化"、"预警等级"、"循环"等）匹配到对应模板。

- **初始化函数**：验证 AppKey/SecretKey 正确传递
- **预警等级计算**：根据子策略命中条数验证等级判定（红/黄/蓝/绿），含混合计数场景和空字符串安全处理
- **循环函数**（股东公司/持股公司）：
  - 循环终止条件、下标递增
  - **格式兼容性**（v2.0）：JSON数组、英文逗号分隔、中文逗号分隔三种 keyno 列表格式
  - **单元素**（v2.0）：单公司单次迭代后结束
- **数据合并函数**：验证多轮循环数据正确合并
  - **跨迭代累加**（v2.0）：如 2+3=5 的失信计数累加
  - **null/空安全**（v2.0）：临时字段为空时不崩溃
- **对象取值函数**（企业/个人/关联企业）：
  - 验证字段拆解正确
  - **空数组输入**（v2.0）：验证子策略跳过而不崩溃
  - **null 输入**（v2.0）：安全处理
  - **多对象数组**（v2.0）：仅取第一个元素
- **资质等级计算**：验证降级档位计算（特4→一3→二2→三1）

#### 3.3 决策流分支覆盖用例（v2.0 增强）

根据场景策略 sheet 中的策略，生成路由覆盖用例：

- 按业务类型（借款/债券/委贷/投资/非融）× 企业类型（国企/民营）组合
- 验证正确的子策略被触发
- 验证路由字段正确传递

**DEFAULT/异常分支路径**（v2.0）：

| 场景 | 入参 | 预期行为 |
|------|------|---------|
| 无效企业类型 | S_S_ENTERPRISETYPE=外资 | 走 DEFAULT 分支 |
| 未知业务场景 | C_S_BIZSCENARIO=未知类 | 走 DEFAULT 分支 |
| 投资类-资管产品 | bizType=投资+资管标识 | 走投资类子策略 |
| 国企借款人跳过个人子策略 | 企业类型=国企 | PLEP 规则集不触发 |

#### 3.4 策略预警等级覆盖用例

基于 alertLevelJudgment 规则集的 4 条规则生成：

| 规则 | 条件 | 预期等级 |
|------|------|---------|
| MAIN01 | 极高风险>0 或 高风险>0 或 中风险≥3 | 红色预警 |
| MAIN02 | 高风险=0 且 0<中风险<3 | 黄色预警 |
| MAIN03 | 高风险=0 且 中风险=0 且 低风险>0 | 蓝色预警 |
| MAIN04 | 高风险=0 且 中风险=0 且 低风险=0 | 绿色预警 |

需构造使子策略命中条数满足各等级条件的组合用例。

#### 3.5 跨子策略综合场景（v2.0）

模拟实际业务中多条规则同时命中的场景，验证三个子策略（企业/个人/关联企业）的联合预警等级：

- **多规则同时命中**：企业子策略命中1条高风险 + 个人子策略命中1条中风险 → 验证综合等级为红色
- **边界组合**：企业子策略3条中风险 + 个人/关联无命中 → 验证综合等级为红色（中风险≥3）
- **SAFE 基线**（v2.1 增强）：所有参数设为安全值（含常用字段默认值），三个子策略均无规则命中 → 验证绿色预警
- **混合业务类型×企业类型**：6 种组合各生成代表性综合用例
- **10 条 SAFE 基线**：5 种业务类型 × 2 种企业类型，填充完整安全参数

## 用例设计原则

1. **每条规则至少 1 正 1 反**：确保命中和未命中都被覆盖
2. **阈值规则加边界值**：对含 ≥、≤、>、< 的规则，增加精确阈值用例
3. **id 唯一性**：每条用例使用顺序编号 `TC_001` ~ `TC_N`
4. **S_S_BIZID 必填**：每条用例都必须包含
5. **详情字段结构匹配三方接口出参**：参考三方数据 sheet 中的出参结构构造
6. **路由字段正确设置**：确保用例走到目标规则集（业务类型×企业类型）
7. **组合条件拆分**：对 `且`/`或`/`||` 复合条件，分别测试各子条件
8. **同一规则跨规则集**：部分规则在国企/民营中命中等级不同，需分别生成用例

## 常见问题

### 规则表达式含模糊描述
部分规则描述列包含补充说明（如"对ChangeDate变更日期距今小于90天..."），应优先参考描述列理解规则逻辑，表达式列可能是简化版。

### 多规则共享字段
多个规则可能使用相同的入参字段但要求不同的值。每条用例只需设置目标规则相关的字段，其他字段设为不触发的安全值。

### 循环函数与分页接口
股东公司/持股公司的循环函数依赖企查查 740 等分页接口，测试时需构造多个持股>20%的公司以验证循环逻辑。

### 贷前 vs 贷中
同一规则在贷前（Before）和贷中策略中可能配置不同，需分别为两套策略生成用例。
