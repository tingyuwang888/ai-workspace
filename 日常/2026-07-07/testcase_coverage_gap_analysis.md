## 测试用例生成器覆盖差距分析

对比对象：现有测试报告（V10，236条用例） vs 生成器输出（367条用例）

### 总体结论

生成器在规则覆盖面上已完全覆盖 Excel 中的 139 条规则，且额外覆盖了 11 条 Excel 中未出现的规则（ARGP01-03、EBNP09-11、ENDX13、MAIN01-04）。但在以下 7 类场景上存在覆盖差距。

---

### 差距一：跨子策略综合场景（缺失约 6 条）

Excel 中有一批"综合场景"用例，同时触发企业、个人、关联企业三个子策略中的多条规则，验证多规则同时命中时的整体预警等级。生成器目前按规则逐条生成，没有这种跨子策略组合用例。

| Excel用例 | 场景描述 |
|-----------|----------|
| TC_138 | 综合多规则同时命中 |
| TC_139 | 非融民营资质+中标 |
| TC_140 | 委贷类国企终本 |
| TC_141 | 投资类民营破产 |
| TC_142 | 个人子策略借款民营 |
| TC_143 | 关联企业民营 |

**改进建议**：在生成器中新增"综合场景"模块，按业务类型×企业类型组合，挑选每个组合下 2~3 条代表性规则同时命中，验证整体预警等级。

---

### 差距二：子策略 DEFAULT / 异常分支路径（缺失约 9 条）

Excel 验证了多种"非正常路径"，包括无效企业类型走 DEFAULT、未知业务场景走默认分支、投资类区分资管/非资管、个人子策略在国企借款场景下不进入特定规则集等。生成器的决策流用例仅覆盖了 10 种标准路由（5 业务类型 × 2 企业类型），缺少这些异常/特殊分支。

| Excel用例 | 场景描述 |
|-----------|----------|
| TC_223 | 个人子策略 DEFAULT 分支 |
| TC_224 | C_S_BIZSCENARIO=未知类走默认路径 |
| TC_225 | 投资类×国企×资产管理类特殊分支 |
| TC_226 | 投资类×民营×资产管理类特殊分支 |
| TC_227 | 委贷类×民营：持股/股东循环不触发 |
| TC_228 | 借款类×国企不进入 PLEP 规则集 |
| TC_229 | 无效企业类型(外资) DEFAULT 到 End |
| TC_232 | 投资类×国企 - 非资管类 |
| TC_234 | 投资类×民营×非资管类 |

**改进建议**：从策略配置文件或落地方案 Excel 中解析决策流的 DEFAULT 分支和特殊条件分支，为每种异常路径生成 1 条验证用例。

---

### 差距三：函数格式兼容性与健壮性（缺失约 22 条）

Excel 中函数模块有 33 条用例，生成器仅 25 条。差距主要在以下三类：

**3a. keyno 列表格式兼容性（缺失 4 条）**

股东公司/持股公司循环函数（S000004/S000005）接收的 keyno 列表支持 JSON 数组、英文逗号分隔、中文逗号分隔三种格式。Excel 对每种格式都单独验证，生成器只覆盖了 JSON 数组格式。

| Excel用例 | 场景描述 |
|-----------|----------|
| TC_190 | 股东公司循环 - 英文逗号分隔 |
| TC_191 | 股东公司循环 - 中文逗号分隔 |
| TC_194 | 持股公司循环 - 英文逗号分隔 |
| TC_195 | 持股公司循环 - 中文逗号分隔 |

**3b. 跨迭代数据累加与合并（缺失 4 条）**

Excel 验证了循环函数跨迭代的失信计数累加（如 2+3=5）和详情数组合并，生成器只验证了单次迭代或最终结果。

| Excel用例 | 场景描述 |
|-----------|----------|
| TC_205 | 持股公司循环 - 跨迭代累加(2+3=5) |
| TC_207 | 股东公司数据合并 - 跨迭代累加 |
| TC_210 | 持股公司数据合并 - 跨迭代累加 |
| TC_212 | 极高风险>0（EBNP08 触发） |

**3c. null / 空输入安全处理（缺失约 8 条）**

Excel 对 S000010/S000011/S000012 的空数组、null 输入、多对象数组（仅取第一个元素）分别做了验证，确保策略不崩溃。生成器仅覆盖了正案例和简单反案例。

| Excel用例 | 场景描述 |
|-----------|----------|
| TC_180 | 三个对象取值函数全部空数组输入 |
| TC_196 | S000010 空数组 |
| TC_197 | S000011 空数组（个人子策略跳过不崩溃） |
| TC_198 | S000012 空数组 |
| TC_199 | S000010 多对象数组（仅取第一个） |
| TC_201 | S000002 空字符串输入 |
| TC_202 | S000010 null 输入 |
| TC_203/204 | S000011/S000012 多对象数组 |
| TC_208/211 | S000007/S000008 临时字段 null |

**3d. 预警等级计算混合场景（缺失约 6 条）**

| Excel用例 | 场景描述 |
|-----------|----------|
| TC_185 | HIGH=4, MED=2, LOW=0 混合计数 |
| TC_200 | HIGH=9, MED=7, LOW=0 大量混合 |
| TC_187-191 | 股东公司循环各阶段验证 |

**改进建议**：在函数用例生成逻辑中增加格式兼容性变体（逗号分隔）、跨迭代累加场景、null/空/多元素输入安全测试。

---

### 差距四：复杂规则的条件拆解与 OR 边界（缺失约 30 条）

Excel 中对包含 AND/OR 多条件的复杂规则做了条件拆解——单独验证每个子条件不满足时的不命中行为。生成器目前对每条规则只生成 1 正 + 1 反 + 最多 1 边界，未拆解多条件规则。

典型案例：

**EBNP06（核心资质到期，Excel 10 条 vs 生成器 2 条）**
Excel 额外覆盖了：距到期=91天（不命中）、=30天（恰好命中高风险，≤30含等于）、=0天（已过期，极高风险）、=-5天（过期5天）。

**ELLP06/ARGP06（自诉案件 OR 条件，Excel 各 8~12 条 vs 生成器 3 条）**
Excel 拆解了 OR 条件的两个分支：≥5笔（命中）、单笔≥1000万（命中）、恰好5笔（边界含等于）、4笔且<1000万（不命中）、=999万（不命中）。

**EBGS11（注册资本虚高三条件 AND，Excel 8 条 vs 生成器 2 条）**
Excel 拆解了三个 AND 子条件：注册资本=4999万（第一条件不满足）、实缴比例=5%（<5%不含等于）、成立年限约2年（<3年条件不满足）。

**EBGS05/06（注册资本减少，Excel 合计 13 条 vs 生成器 6 条）**
Excel 覆盖了减少比例=50%（命中高风险）、=49%（不命中高风险，命中中风险）、=30%（恰好命中中风险）、=29%（均不命中）。

**其他缺口较大的规则**：EBNP07（资质状态注销/正常）、EBGP09/10/11（实缴比例和注册资本多条件）、ENDX02/10（>阈值不含等于的边界）、ARGS01/ELLS05（涉案金额边界299万/499万）。

**改进建议**：当规则的表达式包含 AND/OR 多条件时，为每个子条件生成独立的边界用例（AND 拆解：逐一不满足；OR 拆解：分别满足每个分支），并针对">" vs ">="、"<" vs "<="的包含/不包含等于做精确边界。

---

### 差距五：三方数据描述（Excel 236 条 vs 生成器 11 条）

Excel 的 Col15"三方数据"列记录了每条用例需要配置的 mock 数据摘要（如 2003 返回的实缴比例、887 返回的案件信息、征信报告状态等），便于测试人员确认 mock 配置正确。生成器仅在 params 中隐含了三方数据的 searchKey，未显式生成三方数据描述字段。

**改进建议**：在生成器输出中增加 `thirdPartyMock` 字段，根据规则依赖的数据源自动生成 mock 配置摘要。

---

### 差距六：反向用例的多样性不足（Excel 38 条 vs 生成器 155 条，但部分规则反例不够精细）

虽然生成器的反案例总量（155 条）远超 Excel（38 条），但 Excel 的反案例更加精细——针对同一规则的不同子条件分别构造不命中场景。生成器的反案例通常是"把所有条件都设为不满足"的粗粒度方式。

例如 EBGS11 的 3 条反向用例：Excel 分别测试注册资本不足、实缴比例恰好等于边界、成立年限不满足，而生成器只有 1 条"构造不满足条件"。

---

### 差距七：全 SAFE 基线用例缺少子策略验证

Excel 中 TC_213-221 是"全 SAFE 无风险命中"用例，用于验证 10 种业务类型×企业类型组合下三个子策略均被正确调用且不触发任何规则。生成器虽有 10 条决策流路由用例（TC_FLOW_001-010），但预期结果仅描述了路由到的规则集名称，未明确要求验证"三个子策略均被调用"这一行为。

---

### 改进优先级建议

| 优先级 | 差距类型 | 预估新增用例数 | 实现难度 |
|--------|----------|---------------|----------|
| P0 | 复杂规则条件拆解与 OR 边界 | ~30 条 | 中（需解析 AND/OR 表达式结构） |
| P1 | 函数格式兼容性与健壮性 | ~22 条 | 低（添加固定模板即可） |
| P1 | 子策略 DEFAULT / 异常分支 | ~9 条 | 中（需解析决策流默认分支） |
| P2 | 跨子策略综合场景 | ~6 条 | 低（组合已有规则即可） |
| P2 | 三方数据描述字段 | 影响全部用例 | 低（从数据源映射生成） |
| P3 | 反向用例精细化 | ~15 条 | 中 |
| P3 | 全 SAFE 基线增强 | ~10 条 | 低 |
